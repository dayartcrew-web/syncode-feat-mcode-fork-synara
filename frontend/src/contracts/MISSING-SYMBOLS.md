# MISSING-SYMBOLS — `@t3tools/contracts` shim type-hole worklist

> **Input for T3 (Tier 1: RPC registry + param types) and T4 (Tier 2: domain-event union).**
> Generated by vendoring MCode `apps/web` onto the T1 contracts shim and running
> `tsc --noEmit`. Every entry below is a symbol MCode imports from
> `@t3tools/contracts` that the shim does not yet export.

The shim (Tier 0) exports 26 ts-rs types + 13 branded IDs + runtime guards.
MCode's `apps/web` imports **213 additional symbols** — these are the type
holes T3/T4 must fill. The deduplicated count (213) is far smaller than the
raw import-line count (~1500+) because many symbols are imported across
dozens of files.

## Methodology

1. Vendored MCode `apps/web/src` → `frontend/src` (overlay, preserving T1's
   `contracts/` + `types/`).
2. Vendored MCode `packages/shared/src` → `frontend/src/shared` (so
   `@t3tools/shared/*` subpath imports resolve).
3. Ran `tsc --noEmit`; captured full output.
4. Extracted unique symbol names from `TS2305` (`has no exported member 'X'`)
   and `TS2724` (`has no exported member named 'X'`) errors whose module is
   `@t3tools/contracts`.
5. Grouped by likely domain (heuristic: name prefix + Syncode contracts crate
   module layout — see `crates/syncode-contracts/src/`).

## How T3/T4 should use this

- **T3 (Tier 1 — RPC)**: every `*Input`, `*Result`, `*Options`, `*Event`,
  `*StreamItem`, `Ws*`, `Orchestration*` symbol maps to an RPC method param
  or push-channel payload. Define them in `src/contracts/rpc.ts`.
- **T4 (Tier 2 — events)**: every `*Event`, `*StreamEvent`, `*UpdatedPayload`,
  `OrchestrationShellStreamItem` joins the discriminated union. Define in
  `src/contracts/events.ts`.
- **Constants** (`UPPER_CASE`): mirror MCode's `@t3tools/contracts` constant
  values verbatim (find them in `packages/contracts/src/` in MCode source).
- Cross-reference against Syncode's `crates/syncode-contracts/src/lib.rs` —
  many of these types already exist server-side and need a ts-rs generator
  entry, not a hand-written stub.

## Effect shape-divergence (T5 scope, NOT contracts)

These are `effect` API mismatches between MCode's custom fork
(`pkg.pr.new/Effect-TS/effect-smol/effect@8881a9b`) and stable `effect@3.x`.
T5 strips effect from the frontend; do NOT chase these in T3/T4.

- `Schema.Literals` → stable has `Schema.Literal` (rename)
- `Schema.fromJsonString` → removed in stable
- `Schema.isMaxLength` → stable has `Schema.maxLength`
- `Cause.hasInterruptsOnly` → stable has `Cause.isInterruptedOnly`
- `Scope.provide`, `Scope.Closeable` → API drift
- `effect/unstable/rpc`, `effect/unstable/socket/Socket` → ambient-stubbed in
  `src/syncode-vendor-augmentations.d.ts` (fork-only subpaths)
